Сервис по проверке работы устройства на обсерватории.
Существует 3 возможных типа взаимодействия:
1) запостить в базу данных данные об устройстве
POST реквест по адресу http://host:8001/post-status/
в JSON формате
{
    "obs":"TST",
    "dev": "pos1",
    "date0": 1674210980, #sec
    "date1": 1674210976000, #ms
    "filename": "record2023",
    "md5": "6bb99bff13690e99cecb8be0768b0d8a"

}

2) получить из базы данных данные об устройстве
GET реквест по адресу http://host:8001/get-status/
в JSON формате
{
    "obs":"IRT",
    "dev": "pos1",
    "date0_from": 1674210976,
    "date0_to": 1674210977,
    "date1_from": 1674210976000,
    "date1_to": 1674210976000
}
ответом будет получен словарь, где каждый ключ будет иметь массив
с данными в указанном временном периоде
{"obs": ["IRT", "IRT"], "date0": [1674210976, 1674210977], "dev": ["POS1", "POS1"],
"date1": [1674210976000, 1674210976000], "filename": ["record2023", "record2023"],
"md5": ["5fd9b7478d8bf59e13b1aa32fa7188ee", "5fd9b7478d8bf59e13b1aa32fa7188ee"]
}

3) получить из базы данных последнюю запись по станции и устройству
GET реквест по адресу http://host:8001/get-last-status/
в JSON формате
{
    "obs":"IRT",
    "dev": "pos1"
}

ответ - последняя запись до указанным ключам 'obs' и 'dev'
{"obs": ["IRT"], "date0": [1674210979], "dev": ["POS1"], "date1": [1674210976000],
 "filename": ["record2023"], "md5": ["5fd9b7478d8bf59e13b1aa32fa7188ee"]
 }

для п2 и п3 - если данных нет за указанный период, будет возвращен пустой словарь {}
